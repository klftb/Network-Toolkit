# Network Toolkit 软件使用介绍

Network Toolkit 是一套开箱即用、涵盖多种常用网络调试、检测与规划计算功能的轻量级平台工具。你可以将本工具应用于网络连通性排查、故障分析、拓扑规划以及常用文书的生成等场景。

## 1. 入门指引

打开工具主页后，您将看到**仪表盘（Dashboard）**，在这里我们集中展示了所有的工具入口。
- **左侧导航栏**可以用来在各个功能间快速切换。支持水平折叠功能，您可以点击侧边栏下方（或仪表盘旁的缩进按钮）隐藏文字说明以扩大工作区。
- **最近使用**模块会显示您最近打开的 3 个工具，方便您在频繁使用的网络排查步骤间来回切换。

## 2. 核心功能模块解析

### 2.1 连通性分析 (Ping / TCPing)
此工具为排查服务器是否在线的首选。
- 输入目标 IP 或域名，即可发起 ICMP Ping 或 TCPing 测试。可以直观看到延迟波动及丢包率。

### 2.2 路由追踪 (Tracert / MTR)
当服务器虽然在线、但感觉到网络卡顿或部分丢包时使用。
- 程序会探测访问目标服务器途经的所有路由器与网关，帮助您分析具体的故障或高延迟是在哪一个网络节点发生的。

### 2.3 IP / 端口扫描 (Port Scanner)
当您需要检查某台服务器上是否开放了对应的服务（例如 80(HTTP)、443(HTTPS)、22(SSH) 等）时使用。
- 支持批量连续端口扫描，以直观的形式列出成功建立连接的端口及其可能对应的服务类型。

### 2.4 子网掩码计算 (Subnet Calculator)
网络规划与实施的重要计算辅助。
- 给定一个 IP 地址和掩码位数（CIDR），工具会自动为您计算出：网络地址、广播地址、可用 IP 地址数量以及每一个可用 IP 的起止范围。

### 2.5 公网 IP 查询 (Public IP)
快速了解本地宽带或代理网络当前的出口 IP 地址状态。
- **融合双重源测速**: 国内环境节点升级为 **`www.ip38.com`**，彻底解决了之前旧节点超时、无响应或域名失效等难题。
- **防爬防护穿透**: 服务端使用真实的浏览器 `User-Agent` 头部伪装，防止被远程防爬策略拦截、拒绝（避免了 403 错误）。
- **智能正则抓取**: 采用高效的正则表达式直接锁定网页中的 HTML 下载落地页路径，即使出口处于复杂内网NAT环境下，也能够瞬间、精准地提取到真实的物理出口 IP。

### 2.6 DNS 公网地址 (DNS Tool)
测试所设定的各大公共 DNS 服务器（如 114.114.114.114, 8.8.8.8 等）对特定域名的解析速度。
- 用它来挑选延迟最低、最适合您当前网络环境的自建或备用 DNS 服务器。

### 2.7 文本批处理 (Batch Generator) 与 配置模板
这两项提供给运维网络工程师的高级辅助能力：
- 支持网络设备的 CLI 命令行命令批处理化生成。
- 大量同质化的 IP 列表或变量替换，可以在批处理模块一键完成。

### 2.8 白板草图 (Excalidraw)
无需开启庞大的 Visio等应用，直接在网页中利用离线白板画出临时的网络拓扑图、架构草图，并可以直接导出保存为图片。

### 2.9 Wi-Fi 信号扫描与信道分析器 (Wi-Fi Analyzer)
专为网络工程技术人员排查现场无线局域网设计的重磅利器。
- **跨平台中文原生适配**: Windows 平台采用自研的 Buffer 解码和解析模块，借助 `iconv-lite` 对 GBK 字节流进行高精解码。由于完美支持了“信号”、“信道/信 道”、“无线电类型”、“传输速率”等中英文双语 `netsh` 关键词，从此杜绝了中文 Windows 系统因编码或关键字多义造成扫描为空的情形。
- **信道与频谱图表绘制**: 提取出来的无线网络能够在前端折线图、雷达图和极区图中即时生成信道中心频率的散布图形。
- **非连接空缺智能反算**: 修复了以往在断开 Wi-Fi 连接或未处于连通状态时，无法读取频率、检测不到卡片和频带的顽障。程序增加纯前端与后端双重换算机制：
  - **2.4 GHz**: 识别到信道 `1 - 14` 时，自动反推得出所属 2.4GHz 频带，并按公式 `2407 + 信道号 * 5 (MHz)` 反算其中心频率。
  - **5 GHz**: 识别到大信道号 `> 14` (如 `36, 40, 149, 157` 等) 时，立刻反推得出所属 5GHz 频带高频，并按公式 `5000 + 信道号 * 5 (MHz)` 换算标称频率。
  这确保了即使是处于离线断开、没有任何 Wi-Fi 连通的物理环境下，所有 2.4G 及 5G 附近的基站信息与各信道热力强度折线也能 100% 畅通重现。
- **获取基站定位 (Geolocation Georeference)**: 结合 HTML5 浏览器的 `navigator.geolocation` 高精度地理定位系统。通过点击“获取基站定位”按钮，即可拉取当前网络终端的高精度经度、纬度座标、米级精确范围及时间戳，为你在分析无线网卡地理物理覆盖和信号测量时，建立非常标准的现场历史排障地图定位基线。

### 2.10 文件传输 (FTP / SFTP / TFTP)
针对网络设备的固件刷写或配置备份等场景：
- **FTP / SFTP 客户端**: 提供图形化的目录浏览与文件上传下载功能。在任务列表中，您可以清楚看到实时的进度条与传输速度。支持对大文件传输进行**暂停**与**继续**，或者在卡死时直接**终止**。
- **TFTP 服务端**: 专门提供轻量化 UDP 传输服务端。开启服务后即可在局域网内接受来自网络设备的固件下载或日志上传请求。状态面板可以实时监控并发连接进度和速度，并且提供**强制中断**能力来清理死连接。
- **后台持久化保持**: 当您在此界面建立连接并开始传输后，您可以放心切换到其它工具栏去执行别的排障操作，您的传输任务会安全地在后台继续进行，绝不会因为切出界面而断开或丢失任务状态。
## 3. 本地安装异常排除指南 (针对开发者)

如果你将代码下载到了本地并希望在本地运行，请注意：
1. 确保系统预装 Node.js 环境。
2. Windows 用户如果在使用 `npm install` 时遇到 `enoent Could not read package.json` 错误，这意味着你当前执行命令的控制台所处的目录不正确。
   **解决办法**：请将下载下来的压缩包**完全解压**，然后再在命令行终端使用 `cd` 命令进入解压出来包含有名为 `package.json` 的主目录之后，再执行 `npm install`。
